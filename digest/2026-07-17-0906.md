## Changes

- Fixed a bug where Electric collections using persistence could lose previously saved rows when resuming, causing data to disappear on reload.
- Live queries backed by on-demand Electric data no longer get stuck in a loading state on some native platforms when the refresh step takes too long to cancel.
- Query collections can now accept several more built-in query behaviors (like refetch-on-focus, refetch-on-reconnect, refetch-on-mount, and network mode) directly as top-level options.
- Fixed a bug where query collections could keep stale internal listeners around after a subset was unloaded or cleaned up mid-request.
- Internal cleanup of query collection ownership-tracking code with no change in behavior for users.
- Fixed a regression where queries on collections with a generic row type failed to typecheck, including when using them as join sources.
- Fixed a bug where explicitly setting a live query's garbage-collection time to zero (to keep it alive indefinitely) was ignored and a default timeout was used instead.
