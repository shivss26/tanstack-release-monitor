## Changes

- Fixed a crash that could happen in multi-step forms when a group-level validator reported an error for a field that hadn't rendered yet.
- Fixed forms getting stuck unable to resubmit after a field's blur validation had previously failed, even once the underlying issue was corrected.
- Fixed field-level error tracking not being set up when an asynchronous validator returned errors, so those errors now show up properly.
- Fixed a race condition where the "validating" state could get stuck on when several linked fields triggered async checks at the same time.
- Added a clearer, newer way to subscribe to form state changes, with the older approach kept working but now marked as deprecated.
- Fixed a TypeScript build issue and improved type checking for one of the supported validation libraries.
- Removed an unnecessary required dependency from the Next.js and Remix integrations.
